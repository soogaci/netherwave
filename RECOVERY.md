# Восстановление функционала

Если ты потерял обновлённый код и поднимаешь проект с нуля — ниже что **уже есть в репозитории** и что можно внедрить заново.

## Что уже реализовано в коде

- **Авторизация** — Supabase Auth, страница `/auth`, callback
- **Профиль** — `/profile`, редактирование, настройки, вкладки: посты / Feels / репосты
- **Лента People/Music** — хаб, посты с лайками и комментариями
- **Feels** — лента видео, загрузка в `feel-videos`, описание, лайки, комментарии
- **Репосты Feels** — `feel_reposts`, кнопка «Репост», вкладка «Репосты» в профиле
- **Отправить филс другу** — выбор подписок и отправка в чат (как вложение)
- **Чаты и сообщения** — список чатов, переписка, вложения
- **Уведомления** — страница `/notifications`
- **Посты с фото** — миграция 010, поля `has_photo`, `photo_urls`
- **Миграции БД** — все 15 файлов в `supabase/migrations/` (в т.ч. репосты постов, Feels, чаты, RLS)

## Что может быть не доведено или потеряно

- **Репосты постов People/Music** — таблица `post_reposts` (миграция 014) есть, в коде ленты/профиля репостов постов пока нет
- Любые доработки UI/логики, которые ты делал в потерянной версии

## Что сделать, чтобы локально было всё актуально

1. **Окружение** — в корне создай `.env.local` с ключами того Supabase-проекта, с которым хочешь работать (продакшен или отдельный тестовый):
   ```
   NEXT_PUBLIC_SUPABASE_URL=https://xxx.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=...
   ```

2. **Схема БД** — на этом проекте Supabase должны быть применены все миграции. Варианты:
   - `npx supabase login` → `npm run db:link` → `npm run db:push`, или
   - вручную выполнить все SQL из `supabase/migrations/` по порядку (см. `supabase-setup.md`).

3. **Запуск** — `npm install && npm run dev`. Локально приложение будет работать с выбранной БД, данные и схема будут актуальными.

## Что внедрить заново

Напиши, что именно ты нарабатывал и чего не хватает (например: репосты постов People/Music, донаты, ещё один тип поста и т.д.) — можно по пунктам восстановить и дописать код.
