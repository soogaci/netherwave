# Как настроить деплой на feelreal.ru

Сейчас папка привязана к проекту **netherwave-main**. Если твой сайт должен открываться по **feelreal.ru**, возможны два варианта.

---

## Вариант 1: У тебя уже есть проект в Vercel с доменом feelreal.ru

### Шаг 1. Узнай название этого проекта

1. Зайди на [vercel.com](https://vercel.com) и войди в аккаунт.
2. На главной открой список проектов.
3. Зайди в каждый проект по очереди: **Settings** → вкладка **Domains**.
4. Найди проект, у которого в списке доменов есть **feelreal.ru** (или www.feelreal.ru). Запомни **название проекта** (например `feelreal` или `feelreal-app`).

### Шаг 2. Привяжи эту папку к этому проекту

В терминале (в папке проекта):

```bash
cd /Users/user/netherwave-main
rm -rf .vercel
npx vercel link
```

Дальше отвечай так:

- **Set up "~/netherwave-main"?** → **yes**
- **Which scope?** → выбери свой аккаунт (ivan's projects и т.п.)
- **Link to existing project?** → **yes**
- **What’s the name of your existing project?** → введи **точное название** проекта с feelreal.ru (то, что запомнил в шаге 1)

Если проект найден, появится сообщение вроде `Linked to ...`. После этого:

```bash
npx vercel --prod
```

Деплой пойдёт в выбранный проект, и изменения будут на feelreal.ru.

---

## Вариант 2: Отдельного проекта с feelreal.ru нет — хочешь использовать текущий

Тогда нужно повесить домен **feelreal.ru** на проект **netherwave-main**:

1. Зайди на [vercel.com](https://vercel.com) → открой проект **netherwave-main**.
2. **Settings** → **Domains** → **Add**.
3. Введи **feelreal.ru** и при необходимости **www.feelreal.ru**.
4. Vercel покажет, какие записи добавить у регистратора домена (A или CNAME). Добавь их в панели, где купил feelreal.ru.
5. После проверки DNS деплои из этой папки будут открываться по feelreal.ru.

Папку заново привязывать не нужно — оставляй `npx vercel --prod` как есть.

---

## Если не получается найти проект или домен

- **Не видишь проект с feelreal.ru** — возможно, домен привязан к другому аккаунту/команде в Vercel. Зайди под тем аккаунтом или переключи команду (сверху в Vercel).
- **При `vercel link` не показывается нужный проект** — проверь, что выбран правильный scope (аккаунт/команда). Список проектов зависит от выбранного scope.
- **Домен уже занят другим проектом** — в том проекте в **Settings → Domains** удали feelreal.ru, потом добавь его в нужный проект (netherwave-main или тот, к которому привязываешь папку).

---

## Деплой после настройки

Всегда из папки проекта:

```bash
cd /Users/user/netherwave-main
npx vercel --prod
```

Локальные переменные Supabase в `.env.local` не трогай; для продакшена настрой **Environment Variables** в Vercel у того проекта, куда деплоишь.
